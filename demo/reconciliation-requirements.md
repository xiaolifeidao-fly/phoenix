# 资金出入账对账工作台需求文档

## 1. 文档说明

本文档基于当前 HTML 原型整理，用于后续产品评审、后端建模、接口设计和前端落地。

当前原型文件：

- `demo/index.html`

核心目标：

- 以日期区间为入口，展示人工维度、上游维度、出入账、账户状态。
- 人工维度、上游维度为系统汇总数据，不允许页面手动修改。
- 出入账基于流水明细，可新增、修改、删除，并支持按类目汇总。
- 账户状态分为系统计算与人工录入，人工录入支持按日期新增，并对系统计算结果做差异预警。

## 2. 已确认口径与建议优化点

以下为当前已确认的业务口径，以及建议在落地时保留的扩展能力。

### 2.1 定时任务时间

已确认：

- 每天 `00:10:00` 执行。
- 按业务时区 `Asia/Shanghai` 计算账期。
- 统计日期为前一自然日。

建议：

- 每天 `00:10:00` 生成前一自然日的快照或汇总。
- 支持按日期手动重跑，便于修复历史数据。

### 2.2 人工维度“出款金额 / 代付金额 / 人工总出款”的公式

当前已明确：

- 大户出款金额 = 大户单价 * 大户任务数量。
- 小户出款金额 = 小户单价 * 小户任务数量。
- 其他用户存在出款金额。
- 失败数量不参与金额计算。
- 代付金额 = 出款金额 * 统一出款代付比例，只对 RMB 生效。
- 人工总出款 = 出款金额 - 代付金额。

建议：

- 将页面口径统一命名为：
  - 任务数量
  - 应出款金额
  - 代付金额
  - 实际出款金额
  - 当前余额
  - 人工总出款

### 2.3 RMB 与 U 的换算口径

当前页面为 RMB 和 U 独立展示。

已确认：

- 如果数据源同时存在 RMB 和 U，则优先使用原始字段。
- 如果只有 RMB，需要按汇率折算 U。

建议：

- 优先保留 RMB 与 U 两套原始口径。
- 新增 `exchange_rate_snapshot` 或在汇总表中保存当日使用汇率，避免历史数据随汇率变化。
- 定时汇总任务使用统计日期对应的有效汇率。

### 2.4 大户、小户名单和单价维护

需要定义：

- 哪些用户属于大户。
- 哪些用户属于小户。
- 大户和小户的计费单价。
- 配置生效时间。
- 修改历史是否保留。

建议：

- 建立用户分层与单价配置表。
- 配置按生效日期版本化，历史统计按当时配置计算。

### 2.5 上游小费比例配置

已确认：

- 小费金额 = 消费金额 * 小费比例。
- 小费比例为全局配置。

建议：

- 建立小费比例配置表。
- 支持生效日期，历史汇总按当日有效配置计算。
- 字段命名使用 `tip_ratio`，避免误写成消费比例。

### 2.6 账户状态系统计算表与人工录入表

需求描述中提到：

- 出入账和账户状态都有明细，所以无需打快照。
- 账户状态目前缺失数据表，需要系统计算表和人工表。

建议理解：

- 账户状态不做额外快照汇总表。
- 但需要保存每日系统计算结果明细表。
- 同时需要保存每日人工录入结果表。
- 两张表按日期、账户主体进行对比。

建议采用：

- 账户状态系统计算结果每日 `00:10:00` 固化一份，统计日期为前一自然日。
- 差异预警建议落库成待跟进任务，同时页面展示。

### 2.7 差异预警阈值

当前需求：

- 人工录入与系统计算相关字段差异超过 `5%` 时，需要警示差异点。

建议：

- 阈值做成可配置项，默认 `5%`。
- 差异计算公式：
  - `差异率 = abs(人工值 - 系统值) / abs(系统值)`
  - 系统值为 `0` 时，如果人工值不为 `0`，按 `100%` 差异处理。
- 预警字段至少包括日期、账户主体、字段、系统值、人工值、差异率、处理状态、处理人、处理备注。

## 3. 页面原型说明

### 3.1 页面入口

页面顶部提供日期区间：

- 开始日期
- 结束日期

所有模块均按该日期区间展示。

### 3.2 维度核算模块

维度核算模块由两部分组成：

- 人工维度
- 上游维度

两者在同一行并列展示，中间保留间距，作为同一个整体模块。

模块底部展示系统计算：

- 当前人工总出款
- 上游应收
- 总计利润

公式：

- 总计利润 = 上游应收 - 小费金额 - 当前人工总出款

### 3.3 人工维度

人工维度只读展示，不允许页面修改。

展示字段：

- 类目
- 大户 RMB
- 大户 U
- 小户 RMB
- 小户 U
- 其他用户 RMB
- 其他用户 U

当前类目：

- 任务数量
- 需出账金额
- 代付金额
- 总出款
- 当前余额
- 人工总出款

### 3.4 上游维度

上游维度只读展示，不允许页面修改。

展示字段：

- 类目
- 真人 RMB
- 真人 U
- 其他社区 RMB
- 其他社区 U

当前类目：

- 充值金额
- 返点金额
- 消费金额
- 小费金额
- 应收金额

公式：

- 应收金额 = 充值金额 - 返点金额
- 小费金额 = 消费金额 * 小费比例

### 3.5 出入账

出入账支持两个视图：

- 汇总
- 明细

汇总视图：

- 只读。
- 按类目汇总。
- 展示金额 RMB、金额 U、手续费 RMB、手续费 U。

明细视图：

- 支持新增。
- 支持修改。
- 支持删除。
- 只有切换到明细视图时，才允许新增、修改、删除。

新增方式：

- 点击新增后弹窗录入。
- 不直接在表格中插入默认行。

出入账新增字段：

- 日期
- 类型：入账、出款
- 类目：下拉选择
- 金额 RMB
- 金额 U
- 手续费 RMB
- 手续费 U
- 备注

出入账类目：

- 入账：
  - 社区入账
  - 其他入账
- 出款：
  - 大户出款
  - 小户出款
  - 其他用户出款
  - 服务器出款
  - 其他出款

### 3.6 账户状态

账户状态分为两个视图：

- 系统计算
- 人工录入

账户状态纵向为账户主体：

- AK
- 社区
- 代收方

社区为动态主体，可以新增。

横向字段：

- 欠款金额 RMB
- 欠款金额 U
- 账户余额 RMB
- 账户余额 U

系统计算视图：

- 只读。
- 展示系统计算出来的账户状态。

人工录入视图：

- 支持按日期新增。
- 支持修改。
- 新增时通过弹窗录入。
- 新增社区时，可填写新的社区名称。

人工录入新增字段：

- 日期
- 账户主体
- 新增社区名称
- 主体类型
- 欠款金额 RMB
- 欠款金额 U
- 账户余额 RMB
- 账户余额 U

差异预警：

- 人工录入与系统计算按日期、账户主体、字段进行对比。
- 差异超过 `5%` 时展示预警。
- 预警内容包括：
  - 日期
  - 账户主体
  - 字段
  - 系统值
  - 人工值
  - 差异百分比
  - 后续人工跟进提示

## 4. 数据处理总原则

### 4.1 定时任务

每天凌晨 `00:10:00` 定时执行统计任务，时区为 `Asia/Shanghai`，统计日期为前一自然日。

任务内容：

- 汇总人工维度数据。
- 汇总上游维度数据。
- 计算账户状态系统值。
- 计算差异预警。

### 4.2 快照与汇总原则

出入账：

- 已有流水记账表。
- 不需要额外打快照。
- 页面汇总直接从流水明细按日期区间、类型、类目聚合。

账户状态：

- 需要系统计算表。
- 需要人工录入表。
- 由于两者都是按日期、账户主体记录，可作为明细数据使用。
- 不需要额外快照表。

人工维度：

- 需要建立汇总表。
- 每天定时写入汇总结果。

上游维度：

- 需要建立汇总表。
- 每天定时写入汇总结果。

## 5. 人工维度数据逻辑

### 5.1 数据来源

任务数量来源表：

- `order_sum_record`

### 5.2 用户分层

需要维护用户分层配置：

- 大户用户列表
- 小户用户列表
- 其他用户默认归类

大户、小户、其他用户需要维护不同单价或金额来源。

建议配置字段：

- 用户 ID
- 用户名称
- 用户类型：大户、小户、其他用户
- 单价 RMB
- 单价 U
- 生效日期
- 失效日期
- 状态
- 创建人
- 创建时间
- 更新人
- 更新时间

### 5.3 任务数量统计

大户任务数量：

- 取 `order_sum_record` 的：
  - 待审核数量
  - 失败数量
  - 审核成功数量
- 三者相加。

小户任务数量：

- 取 `order_sum_record` 的：
  - 待审核数量
  - 失败数量
  - 审核成功数量
- 三者相加。

其他用户任务数量：

- 只取审核成功数量。
- 失败数量不参与其他用户任务数量，也不参与金额计算。

### 5.4 金额计算

大户出款金额：

- 大户出款金额 = 大户单价 * 大户任务数量

小户出款金额：

- 小户出款金额 = 小户单价 * 小户任务数量

其他用户出款金额：

- 其他用户存在出款金额。
- 其他用户金额计算仅基于审核成功数量。
- 其他用户单价或金额来源需要在配置表中维护，建议与大户、小户使用同一套用户分层和单价配置。

代付金额：

- 代付金额 = 出款金额 * 统一出款代付比例
- 仅 RMB 生效

人工总出款：

- 人工总出款 = 出款金额 - 代付金额

### 5.5 人工维度汇总表

建议表名：

- `manual_dimension_daily_summary`

建议字段：

| 字段 | 说明 |
| --- | --- |
| id | 主键 |
| stat_date | 统计日期 |
| user_level | 用户分层：big、small、other |
| task_count | 任务数量 |
| unit_price_rmb | 单价 RMB |
| unit_price_u | 单价 U |
| payable_amount_rmb | 应出款金额 RMB |
| payable_amount_u | 应出款金额 U |
| payout_ratio | 出款代付比例 |
| payout_amount_rmb | 代付金额 RMB |
| payout_amount_u | 代付金额 U，默认 0 或按确认口径 |
| actual_out_amount_rmb | 实际出款金额 RMB |
| actual_out_amount_u | 实际出款金额 U |
| current_balance_rmb | 当前余额 RMB |
| current_balance_u | 当前余额 U |
| created_at | 创建时间 |
| updated_at | 更新时间 |

说明：

- `user_level=other` 需要计算出款金额。
- 如果源数据只有 RMB，需要按统计日期有效汇率折算 U。
- 汇率应在汇总结果中留痕，建议增加 `exchange_rate` 字段或关联汇率快照表。

## 8. 配置数据逻辑

### 8.1 小费比例配置

小费比例为全局配置，用于上游维度每日定时计算小费金额。

建议表名：

- `tip_ratio_config`

建议字段：

| 字段 | 说明 |
| --- | --- |
| id | 主键 |
| tip_ratio | 小费比例 |
| effective_date | 生效日期 |
| status | 启用、停用 |
| remark | 备注 |
| created_by | 创建人 |
| updated_by | 更新人 |
| created_at | 创建时间 |
| updated_at | 更新时间 |

规则：

- 定时任务按统计日期匹配有效的小费比例。
- 同一统计日期只能命中一条启用配置。
- 小费金额 = 消费金额 * 小费比例。

### 8.2 汇率配置

当数据源只有 RMB 时，需要按汇率折算 U。

建议表名：

- `exchange_rate_snapshot`

建议字段：

| 字段 | 说明 |
| --- | --- |
| id | 主键 |
| rate_date | 汇率日期 |
| currency_from | 原币种，默认 RMB |
| currency_to | 目标币种，默认 U |
| exchange_rate | 汇率 |
| source | 汇率来源 |
| created_at | 创建时间 |
| updated_at | 更新时间 |

规则：

- 定时任务按统计日期使用当日有效汇率。
- 汇总表建议保存使用的汇率或汇率快照 ID，保证历史结果可追溯。

## 10. 定时任务设计

### 10.1 任务名称

建议：

- `daily_reconciliation_summary_job`

### 10.2 执行时间

每天 `00:10:00` 执行，时区为 `Asia/Shanghai`。

### 10.3 执行步骤

1. 读取统计日期，统计日期为当前执行日的前一自然日。
2. 汇总人工维度，写入 `manual_dimension_daily_summary`。
3. 汇总上游维度，写入 `upstream_dimension_daily_summary`。
4. 计算账户状态系统值，写入 `account_status_system_daily`。
5. 读取同日人工录入账户状态。
6. 计算差异预警。
7. 写入或更新 `account_status_diff_alert`。
8. 记录任务执行日志。

### 10.4 重跑机制

建议支持按日期重跑：

- 重跑人工维度汇总。
- 重跑上游维度汇总。
- 重跑账户状态系统计算。
- 重算差异预警。

建议采用幂等写入：

- 同一日期、同一维度、同一主体唯一。
- 重跑时更新已有记录，不重复插入。

## 12. 权限与操作约束

人工维度：

- 只读。

上游维度：

- 只读。

出入账：

- 汇总视图只读。
- 明细视图允许新增、修改、删除。
- 新增通过弹窗。

账户状态：

- 系统计算只读。
- 人工录入允许新增、修改。
- 新增通过弹窗。
- 动态社区通过人工录入新增时创建账户主体。

配置：

- 大户、小户、其他用户单价、小费比例、汇率、账户主体配置需要单独权限。

## 13. 验收标准

1. 页面可以按日期区间查询。
2. 人工维度只读展示，数据来自每日汇总表。
3. 上游维度只读展示，包含充值金额、返点金额、消费金额、小费金额、应收金额。
4. 出入账汇总按类目展示。
5. 出入账明细支持弹窗新增、行内修改、删除。
6. 出入账新增时类目为下拉选择，且随入账/出款类型变化。
7. 账户状态系统计算按账户主体纵向展示。
8. 账户状态横向展示欠款金额和账户余额。
9. 社区账户主体支持动态新增。
10. 账户状态人工录入支持按日期弹窗新增。
11. 人工录入和系统计算差异超过 5% 时展示预警。
12. 每天 `00:10:00` 按 `Asia/Shanghai` 时区定时生成前一自然日的人工维度、上游维度和账户状态系统计算数据。
13. 定时任务支持指定日期重跑。
